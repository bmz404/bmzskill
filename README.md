# bmzskill

把音乐创作、测歌、宣发和变现，变成一套可执行的 AI 判断流程。

bmzskill 由自媒体博主「白帽子（AI音乐研究中）」创建，用于帮助音乐创作者、独立音乐人和短视频博主，用 AI 辅助音乐定位、创作、测试、宣发和复盘。

bmzskill 不是一键生成歌曲的按钮，而是一套面向音乐短视频传播的判断流程。它会帮你判断歌词能不能打中听众、视频能不能发布、发布包怎么写、数据反馈说明什么，以及下一步该继续放大还是及时止损。

## 核心流程

```text
/bmz 主入口
  -> /bmz-lyrics-check 歌词质检
  -> /bmz-preflight 发布前体检
  -> /bmz-publish 发布包生成
  -> /bmz-review 发布后数据复盘
```

这个流程对应真实发布路径：

```text
明确问题 -> 检查歌词 -> 发布前判断 -> 生成发布包 -> 看数据决定下一步
```

## 功能模块

| 模块 | 中文名 | 适合什么时候用 | 它会帮你判断什么 |
|---|---|---|---|
| `/bmz` | 主入口 | 你不知道该从哪里开始，或只想直接说需求 | 自动识别你的意图，并带你进入合适模块 |
| `/bmz-lyrics-check` | 歌词质检 | 你已经有歌词、主打段、hook 或几句想法 | 活人感、逻辑、情绪递进、第一句抓人能力、押韵与 flow、短视频传播适配度 |
| `/bmz-preflight` | 发布前体检 | 你已经做好了音乐短视频，准备发布 | 画面、歌词、曲风、音色、字幕、标题、标签是否打向同一类听众 |
| `/bmz-publish` | 发布包生成 | 你准备发视频，但不知道标题、首屏文案钩子、评论和标签怎么写 | 标题、首屏文案钩子、置顶评论、标签是否服务同一个传播目标 |
| `/bmz-review` | 发布后数据复盘 | 你已经发布了作品，想知道下一步怎么改 | 播放、2 秒跳出、5 秒留存、完播、点赞、收藏、评论、转发分别说明什么 |

## 怎么开始

如果你是新手，可以直接输入：

```text
/bmz 新手入门
```

如果你已经有歌词，可以输入：

```text
/bmz-lyrics-check
```

然后把歌词发给它。

如果你已经有待发布的视频，可以输入：

```text
/bmz-preflight
```

然后上传视频或截图，让它做发布前判断。

如果你已经准备发布，可以输入：

```text
/bmz-publish
```

它会帮你整理标题、首屏文案钩子、置顶评论和标签。

如果你已经发布并拿到数据，可以输入：

```text
/bmz-review
```

然后上传后台数据截图，或直接写出播放量、2 秒跳出、5 秒留存、完播率、点赞、收藏、评论、转发等数据。

## 判断原则

bmzskill 不是“爆款保证器”，也不是“真理系统”。它更像一个音乐项目判断器：

- 不承诺每首歌都会火，只帮助你降低盲目投入。
- 不把单次建议当成最终答案，而是鼓励小范围测试、看数据、再迭代。
- 不把 AI 当成替代音乐人的按钮，而是把 AI 用在判断、拆解、整理和复盘上。
- 不只服务 AI 生成音乐，也服务真人创作、传统音乐、demo 测试、短视频宣发和音乐项目复盘。

## 数据怎么看

bmzskill 重点关注这些信号：

- `2 秒跳出`：画面、标题、首屏钩子有没有让人停下来。
- `5 秒留存`：第一句歌词、旋律记忆点、主打段进入速度是否成立。
- `完播`：视频节奏、歌曲长度、画面承接是否拖沓。
- `点赞`：观众是否认可这一条内容。
- `收藏`：歌曲、歌词或情绪是否值得反复听。
- `评论`：有没有触发故事、共鸣、求歌名、求完整版、求上架。
- `转发`：这条内容是否能替观众表达某种关系或情绪。

## 判断标准

bmzskill 的判断标准来自真实音乐短视频案例观察、歌词质量规则和短视频数据复盘方法。它不是绝对真理，而是一套帮助用户更快完成“假设 -> 测试 -> 复盘 -> 再测试”的判断框架。

想了解具体依据，可以阅读 [判断标准说明](判断标准说明.md)。
## 发布包规则

`/bmz-publish` 生成发布包时，会遵循这些基本原则：

- 标题和首屏文案钩子尽量不要直接复述第一句歌词。
- 标签不超过 5 个。
- 标签必须包含 `#原创音乐` 和 `#demo`。
- 不使用会让观众先入为主判断“这是 AI 做的歌”的标签。
- 首屏文案钩子服务停留，置顶评论服务互动，标题服务关系场景。

## 适合谁

bmzskill 适合：

- 正在做原创音乐 demo 测试的人。
- 想把歌曲通过短视频推起来的音乐人。
- 已经写了歌词，但不确定能不能传播的人。
- 做了视频，但发布前心里没底的人。
- 发布后有数据，却不知道下一步该换画面、改 hook、换歌还是继续放大的人。

## 不适合谁

bmzskill 不适合：

- 只想一键生成完整成品、不想做测试的人。
- 希望工具直接承诺爆款结果的人。
- 不愿意根据数据调整作品的人。
- 需要法律、版权合同、财务结算等正式专业意见的人。

## 安装方式

将本项目中的 `skills` 文件夹安装到你的 Codex skills 目录后，新开一个任务即可使用。

本项目包含以下技能目录：

```text
skills/
  bmz/
  bmz-lyrics-check/
  bmz-preflight/
  bmz-publish/
  bmz-review/
```

如果你只想先跑通一次完整流程，从 `/bmz 新手入门` 开始即可。
