# bmzskill

 ![version](https://img.shields.io/badge/version-v0.3.0-blue)
![skills](https://img.shields.io/badge/skills-1-black)
[![license](https://img.shields.io/badge/license-CC%20BY--NC%204.0-green)](LICENSE)

把音乐创作、测歌、宣发和变现，变成一套可执行的 AI 判断流程。

bmzskill 由自媒体博主「白帽子（AI音乐研究中）」创建，用于帮助音乐创作者、独立音乐人和短视频博主，用 AI 辅助音乐定位、创作、测试、宣发和复盘。

bmzskill 不是一键生成歌曲的按钮，而是一套面向音乐短视频传播的判断流程。用户只需要记住 `/bmz` 一个入口，它会像主理人一样理解需求，再把任务委派给对应的专家子 Agent，最后统一给出判断和下一步。

## 架构：主 Agent + 子 Agent

bmzskill 从 v0.3.0 起明确采用**主 Agent + 子 Agent** 架构，解决单 skill 内容过多导致的长会话记忆压缩问题：

- **主 Agent（`/bmz`）** 是纯客户经理：只做意图识别、把用户数据填进提示词模板的占位符、spawn 一个全新的专家子 Agent、再把子 Agent 的回复转述给用户。**主 Agent 自己不执行任何专家判断，也不把专家规则堆在自己上下文里。**
- **专家子 Agent** 每次都是独立、干净的新实例：以 `templates/<expert>.prompt.md`（填好用户数据后的参数化提示词）作为系统提示，并加载 `knowledge/<expert>.md` 作为知识库。因为每次都从头满血加载知识，长会话不会压缩任何一次判断的质量。
- 专家流程分为两部分：参数化的**提示词模板**（`templates/*.prompt.md`，主 Agent 填字段后交给子 Agent）与独立的**知识库文件**（`knowledge/*.md`，子 Agent 自己加载）。

```text
用户 -> /bmz 主 Agent（识别+填字段+派发）
                |
                v  spawn 全新子 Agent
       ┌────────┴────────┬──────────┬──────────┬──────────┐
    创作启发          歌词质检    发布前体检   发布包生成   发布后复盘
    .prompt.md        .prompt.md  .prompt.md  .prompt.md  .prompt.md
    +knowledge/        +knowledge/ +knowledge/ +knowledge/ +knowledge/
```

完整发布闭环放在 `bmz/workflows/song-release-loop.md` 里，描述一首歌从主题到数据复盘的连续阶段，以及每个阶段应 spawn 哪个子 Agent。

## 核心流程

```text
/bmz
  -> 创作启发
  -> 歌词质检
  -> 发布前体检
  -> 发布包生成
  -> 发布后数据复盘
```

这个流程对应真实发布路径：

```text
明确问题 -> 检查歌词 -> 发布前判断 -> 生成发布包 -> 看数据决定下一步
```

## 功能模块

| 入口 | 内部流程 | 适合什么时候用 | 它会帮你判断什么 |
|---|---|---|---|
| `/bmz` | 创作启发 | 你只有主题或一句想法 | 目标听众、可写方向、8 句主打段结构 |
| `/bmz` | 歌词质检 | 你已经有歌词、主打段、hook 或几句想法 | 活人感、逻辑、情绪递进、第一句抓人能力、押韵与 flow、短视频传播适配度 |
| `/bmz` | 发布前体检 | 你已经做好了音乐短视频，准备发布 | 画面、字幕、标题、标签、首屏文案钩子和歌词呈现是否打向同一类听众 |
| `/bmz` | 发布包生成 | 你准备发视频，但不知道标题、首屏文案钩子、评论和标签怎么写 | 标题、首屏文案钩子、置顶评论、标签是否服务同一个传播目标 |
| `/bmz` | 发布后数据复盘 | 你已经发布了作品，想知道下一步怎么改 | 播放、2 秒跳出、5 秒留存、完播、点赞、收藏、评论、转发分别说明什么 |

## 怎么开始

如果你是新手，可以直接输入：

```text
/bmz 新手入门
```

如果你已经有歌词，可以输入：

```text
/bmz 检查这段歌词
```

然后把歌词发给它。

如果你已经有待发布的视频，可以输入：

```text
/bmz 检查这条待发布视频
```

然后上传视频或截图，让它做发布前判断。

如果你已经准备发布，可以输入：

```text
/bmz 给这条作品做发布包
```

它会帮你整理标题、首屏文案钩子、置顶评论和标签。

如果你已经发布并拿到数据，可以输入：

```text
/bmz 复盘这条作品数据
```

然后上传后台数据截图，或直接写出播放量、2 秒跳出、5 秒留存、完播率、点赞、收藏、评论、转发等数据。

## 判断原则

bmzskill 不是“爆款保证器”，也不是“真理系统”。它是一个音乐项目判断器：

- 不承诺每首歌都会火，只帮助你降低盲目投入。
- 不把单次建议当成最终答案，而是鼓励小范围测试、看数据、再迭代。
- 不把 AI 当成替代音乐人做歌的按钮，而是把 AI 用在判断、拆解、整理和复盘上。
- 服务所有音乐人，进行原创音乐demo 测试、短视频宣发和音乐项目复盘。

## 数据怎么看

bmzskill 重点关注这些信号：

- `2 秒跳出`：画面、标题、首屏钩子有没有让人停下来。
- `5 秒留存`：第一句歌词、旋律记忆点、主打段进入速度是否成立。
- `完播`：视频节奏、歌曲长度、画面承接是否拖沓。
- `点赞`：观众是否认可这一条内容。
- `收藏`：歌曲、歌词或情绪是否值得反复听。
- `评论`：有没有触发故事、共鸣、求歌名、求完整版、求上架。
- `转发`：这条内容是否能替观众表达某种关系或情绪。

## 判断标准

bmzskill 的判断标准来自真实音乐短视频案例观察、歌词质量规则和短视频数据复盘方法。它不是绝对真理，而是一套帮助用户更快完成“假设 -> 测试 -> 复盘 -> 再测试”的判断框架。

想了解具体依据，可以阅读 [判断标准说明](docs/判断标准说明.md)。
## 发布包规则

`/bmz` 生成发布包时，会遵循这些基本原则：

- 标题和首屏文案钩子尽量不要直接复述第一句歌词。
- 标签不超过 5 个。
- 标签必须包含 `#原创音乐` 和 `#demo`。
- 不使用会让观众先入为主判断“这是 AI 做的歌”的标签。
- 首屏文案钩子服务停留，置顶评论服务互动，标题服务关系场景。

## 适合谁

bmzskill 适合：

- 正在做原创音乐 demo 测试的人。
- 想把歌曲通过短视频推起来的音乐人。
- 已经写了歌词，但不确定能不能传播的人。
- 做了视频，但发布前心里没底的人。
- 发布后有数据，却不知道下一步该换画面、改 hook、换歌还是继续放大的人。

## 不适合谁

bmzskill 不适合：

- 只想一键生成完整成品、不想做测试的人。
- 希望工具直接承诺爆款结果的人。
- 不愿意根据数据调整作品的人。
- 需要法律、版权合同、财务结算等正式专业意见的人。

## 安装方式

### 推荐：一键安装

在终端执行：

```bash
npx -y skills add bmz404/bmzskill -g --all
```

安装完成后，新开一个任务，输入：

```text
/bmz 新手入门
```

即可开始使用。

如果你的电脑提示找不到 `git`，需要先安装 Git，再重新执行上面的命令。

### 备用：使用安装包

如果一键安装失败，可以下载 Release 附件中的安装包：

```text
bmzskill-v0.3.0.zip
```

解压后，把里面的 `bmz` 文件夹复制到你所使用 Agent 的 skills 目录。

### 手动安装：从源码仓库复制

本项目包含以下技能目录：

```text
skills/
  bmz/
```

如果你下载的是 GitHub 的 Code ZIP，注意不要把整个 `bmzskill-main` 文件夹或外层 `skills` 文件夹直接复制到你的 Agent skills 目录。

正确做法是：只复制 `skills` 里面的 `bmz` 文件夹。

如果你只想先跑通一次完整流程，从 `/bmz 新手入门` 开始即可。
