# Suno 技术过时核查清单

核查日期：2026-07-16

用途：记录从内部学习材料中发现的疑似旧版 Suno 术语、功能路径或风险表述。这个文档只用于帮助 bmzskill 在回答技术问题时保持保守、合规、不过度承诺。

## 已确认需要统一改写

### 1. 人声音色功能统一写作 voice

旧材料里可能出现旧版人声称呼。bmzskill 对外统一使用 `voice`。

回答用户时不要说“你去找旧入口”，而是说：

> 如果你需要复用某个稳定人声音色，请优先使用当前版本里的 voice / 声音相关功能；具体按钮位置以你当前界面为准。

### 2. 不提供绕过歌手姓名限制的方法

旧材料中可能出现过“用变体写法绕过知名歌手姓名限制”的经验。这类内容不进入开源方法论。

bmzskill 的统一写法是：

- 不直接调用真实歌手姓名。
- 不指导用户规避平台规则。
- 用“音色、年代、唱法、曲风、制作气质、语言、情绪”描述目标效果。

例如：

```text
不要写：某某歌手同款。
改写为：warm male Chinese R&B vocal, intimate breathy delivery, soft falsetto accents, 2000s Mandarin pop influence.
```

### 3. 不把“AI 音乐”作为公开视频标签

发布包里不建议出现 `#AI音乐`、`#AI歌曲`、`#Suno` 等暴露制作方式的标签。

推荐默认保留：

```text
#原创音乐 #demo
```

其他标签根据作品目标选择，但总数不超过 5 个。

### 4. 不写死平台按钮路径

Suno 等生成平台更新频繁。旧材料中所有“点击某个按钮、进入某个旧页面、选择某个旧入口”的路径，都不能直接搬进回答。

bmzskill 的统一写法是：

```text
功能逻辑可以讲，按钮路径不要写死；如果必须说明路径，要加一句“以当前界面为准”。
```

## 需要继续按当前界面核验

### 1. Cover / Remix / Edit 的入口和命名

可以讲功能边界：

- Cover：更适合保留歌曲骨架，更换演唱或质感。
- Remix / Edit：更适合改变局部、结构或版本。
- Extend：更适合续写已有片段。

但不要承诺每个用户一定能看到同名按钮。

### 2. 上传音频的影响权重

旧材料中如果提到某个固定参数、固定比例或固定最佳值，需要改成区间建议，并提醒用户用两轮测试验证。

### 3. 分轨、MIDI、和弦识别、BPM/调性识别

bmzskill 当前不把音频增强分析作为 V1 默认能力。

原因：

- 自动识别 BPM、调性、和弦、旋律走向存在明显误差。
- 如果判断错误，会误导用户的作曲和复盘。
- 手机端用户也难以稳定使用本地音频分析环境。

统一回答：

> 手机端暂不支持音频分析功能，请使用电脑端。当前 V1 仅做人工听感和短视频适配判断，不提供自动和弦/调性结论。

### 4. 会员、积分、排队、版权条款

这些属于平台商业规则，变化概率高。不要把旧材料中的价格、额度、套餐权益写入固定方法论。

回答时只保留原则：

- 发布前确认当前平台授权规则。
- 商业使用前确认当前账号套餐是否允许。
- 翻唱、改编、采样、使用他人声音时必须额外核查授权。

## 2026-07-16 外部信息提醒

截至本次核查，公开报道显示 Suno 在 2026 年推出的 v5.5 强化了个性化能力，包括 Voices、Custom Models、My Taste 等方向。由于这类功能与会员权益、地区开放、界面入口相关，bmzskill 不把这些功能写成所有用户必然可用的固定步骤，只作为“可检查是否已开放”的能力。

## 不进入开源知识库的内容

- 内部素材文件名。
- 内部素材目录结构。
- 内部素材原文表达。
- 内部称呼。
- 规避平台规则的具体技巧。
- 可能诱导用户侵权的操作步骤。
- 已明显依赖旧界面的按钮路径。
- 未经官方确认的隐藏脚本、隐藏参数、漏洞式提质方法。
- 故意修改歌词拼写、翻译成谐音或改变音频来规避版权保护的做法。
- 用受保护歌曲、受保护歌词或受保护人声做未经授权的上传、翻唱、改编或替换。

## 高风险问题统一回答

如果用户询问如何绕过版权、上传受保护歌曲、处理受保护歌词、规避平台检测或使用隐藏代码，应回答：

```text
这个方向我不能帮你做规避平台规则或版权保护的操作。

更稳的做法是：
1. 使用原创歌词、原创旋律或已授权素材。
2. 如果是翻唱/改编，先确认授权和平台规则。
3. 如果只是想保留某种情绪或质感，可以改用曲风、音色、节奏、编曲和画面方向来描述，而不是复制受保护内容。
```

## bmzskill 的保守回答模板

```text
这个功能平台更新比较快，我不建议把旧版按钮路径当成标准。

我先按功能逻辑给你判断：
1. 你要解决的问题是：{问题}
2. 更适合使用的功能方向是：{voice / cover / extend / edit / prompt / lyrics}
3. 当前界面里你需要寻找的关键词是：{关键词}
4. 如果你的界面没有这个入口，说明功能可能尚未开放、被改名，或与你的账号版本有关。

下一步我可以先帮你把提示词、歌词结构和测试方案写出来。
```
