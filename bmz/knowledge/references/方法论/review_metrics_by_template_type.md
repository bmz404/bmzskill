# 分类型数据复盘指标

来源：A001-A030 A级对标卡 + 七种观众互动冲动模型  
证据等级：B 级方法论，待 S 级后台数据继续校准  
用途：避免 `/bmz-review` 用同一套点赞率、收藏率、评论率、转发率标准判断所有短视频歌曲。

## 核心原则

不同类型的歌曲短视频，核心成功指标不同。

同样是 10 万播放：

- 情歌真人近景：评论和收藏更重要。
- DJ 硬曲：原声使用量和跟拍更重要。
- 入股号 / 求歌名：评论热度重要，但必须警惕歌曲转化弱。
- 制作人 / 音乐人身份：关注和主页点击更重要。
- 玄学许愿歌：转发和收藏更重要。
- 高清 MV / 外景审美：转发和收藏更重要，但要检查是否只火了画面。

所以复盘时必须先判断“这条视频属于哪类模板”，再判断“应该看哪组指标”。

## 1. 情绪流行 / emo / 失恋真人近景

参考案例：A001、A002、A003、A006、A020、A021、A027、A029

优先看：

1. 2 秒跳出率：画面和人物是否让人停。
2. 5 秒留存：第一句歌词是否锁定人群。
3. 评论率：是否有人代入自己的关系。
4. 收藏率：歌曲片段是否值得反复听。
5. 转发率：是否有强共鸣。

不要只看：

- 单条播放量。
- 点赞率。

判断逻辑：

- 评论高 + 收藏高：歌词和歌曲都可能成立。
- 评论高 + 收藏低：话题成立，但歌曲记忆点可能弱。
- 收藏高 + 评论低：歌曲好听，但评论钩子弱。
- 2-5 秒流失高：优先查第一句歌词、vocal 进入和旋律记忆点是否太靠后。

## 2. 歌词流 / 入股号 / 求歌名

参考案例：A007、A010、A014、A018、A026

优先看：

1. 评论率：是否有人参与取名、入股、求完整版、求歌名。
2. 求完整版 / 求上架 / 求歌名评论数量。
3. 收藏率：是否真的有人想留住这首歌。
4. 音乐平台点击或后续转化。
5. 第二条视频复测数据。

不要只看：

- 评论热闹。
- 播放量。

判断逻辑：

- 评论高但收藏低：可能是玩法热，不是歌热。
- 评论高但无人求完整版：可能只是在玩梗。
- 评论高但音乐平台点击低：短视频互动没有转化为歌曲资产。
- 第二轮换包装后仍没人求歌名：优先换歌或重写 hook。

## 3. DJ硬曲 / 纯音乐 BGM / 卡点音乐

参考案例：A030

优先看：

1. 原声使用量。
2. 跟拍增长速度。
3. 收藏率。
4. 转发率。
5. 跟拍画面类型是否多样。

不要只看：

- 评论率。
- 原作者单条视频播放。
- 歌词共鸣。

判断逻辑：

- 单条播放一般，但原声使用量增长：歌曲资产可能成立。
- 点赞高但原声使用量低：视频包装强，音乐使用价值弱。
- 原声使用量高但评论少：正常，BGM 型内容不一定靠评论传播。
- 跟拍画面类型越多，说明这首 BGM 适配面越宽。

## 4. 音乐人身份 / 制作人展示 / 编曲流程

参考案例：A009、A022、A023、A024、A030

优先看：

1. 关注率。
2. 主页点击。
3. 评论区是否认可创作能力。
4. 收藏率。
5. 后续作品连带播放。

不要只看：

- 单条歌是否爆。
- 点赞率。

判断逻辑：

- 关注高：人物和能力被认可。
- 评论区讨论“怎么做的 / 好专业 / 继续发”：账号资产成立。
- 播放高但关注低：内容热，但人没被记住。
- 流程太长导致 5 秒留存低：先缩短流程，把歌曲入口前置。

## 5. 玄学许愿 / 显化歌

参考案例：A019

优先看：

1. 转发率。
2. 收藏率。
3. 评论区许愿数量。
4. 复用场景。
5. 歌曲是否被当作情绪工具。

不要只看：

- 歌词文学性。
- 单条点赞。

判断逻辑：

- 转发高 + 收藏高：功能性强，适合继续做同类主题。
- 评论全是许愿但没人问歌：内容功能强，歌曲资产弱。
- 歌曲被当作背景仪式感：可以往版权 / BGM 方向验证。

## 6. AI MV / 素材氛围 / 电影感切片

参考案例：A008、A011、A012、A015、A016

优先看：

1. 2 秒停留。
2. 收藏率。
3. 转发率。
4. 评论区是否讨论歌，而不只是画面。
5. 音乐平台跳转。

不要只看：

- 画面夸奖。
- 播放量。

判断逻辑：

- 收藏高但没人问歌：可能是画面审美成立，歌曲记忆点弱。
- 转发高：氛围表达有传播价值。
- 评论全是“画面好看”：下一轮要把歌词/歌名/原声入口做强。
- 素材来源不清：版权风险优先级高于数据判断。

## 7. 高清外景 / MV质感

参考案例：A025

优先看：

1. 转发率。
2. 收藏率。
3. 评论区是否讨论“像 MV / 什么时候上线 / 歌名”。
4. 音乐平台点击。
5. 画面和歌曲的绑定程度。

不要只看：

- 画面质感。
- 点赞率。

判断逻辑：

- 转发高 + 收藏高：审美资产成立。
- 画面被夸但没人问歌：画面强，歌曲入口弱。
- 成本高但转化低：不建议继续提高拍摄成本，先改 hook。

## 8. 下沉段子 / 反差剧情 + 演唱

参考案例：A023、A027、A028

优先看：

1. 2 秒跳出率。
2. 评论率。
3. 评论是否从段子转向歌曲。
4. 收藏率。
5. 求歌名 / 求完整版评论。

不要只看：

- 评论热闹。
- 段子播放。

判断逻辑：

- 评论都在聊段子：停留成立，歌曲未必成立。
- 评论开始问歌名：段子成功导向歌曲。
- 收藏低：歌曲片段弱，下一轮要改主打段。
- 高级氛围歌不适合硬套段子。

## 复盘输出要求

每次 `/bmz-review` 必须先输出：

```text
模板类型判断：
最接近案例：
本类型主指标：
本类型不应过度解读的指标：
```

然后再输出常规数据判断。

如果无法判断模板类型，必须写：

```text
模板类型判断：信息不足
需要补充：视频截图 / 录屏 / 发布包 / 歌曲类型 / 评论区截图 / 原声使用量
```

## bmzskill 可复用判断

复盘不是问“数据好不好”，而是问：

> 这类视频本来应该靠什么指标成立？现在数据有没有证明它真的在那个指标上成立？

如果指标和模板错位，就容易误判：

- 拿 DJ 硬曲问评论率，会误判。
- 拿入股号问播放量，会误判。
- 拿真人情歌只看点赞，会误判。
- 拿制作人账号只看单条歌播放，会误判。
- 拿 AI MV 只看画面夸奖，会误判。
