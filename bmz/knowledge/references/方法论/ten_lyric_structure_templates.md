# 10种歌词结构模版

来源：bmzskill 歌词扩写结构库  
证据等级：B 级  
用途：当用户已经有主打段、副歌、Hook 或 8 句歌词，并要求扩写完整歌词时，用于先选择完整歌曲结构，再进入歌词扩写。

## 使用原则

完整歌词不是把主打段前后硬塞几段词，而是先确定歌曲结构。

结构会影响四件事：

- 听众第一次听到副歌的时间。
- 情绪是快速爆发，还是慢慢铺垫。
- 歌词重复密度是否适合短视频传播。
- 完整版是否能支撑音乐平台收听。

当用户说“帮我写完整歌词”“根据这段副歌写完整歌”“把这段 Hook 扩写成整首”“写 verse / bridge / outro”时，必须先展示 10 个结构选项，让用户回复数字编号。

如果用户不想选，可以根据目标默认推荐：

- 短视频传播优先：2. 洗脑病毒式
- 情绪叙事优先：3. 氛围渐进式
- emo 流行 / 旋律说唱：4. 能量爆发式 或 9. 插段变奏式
- 对唱 / 双视角：6. 对话辩论式
- 电影感 / 长叙事：1. 长篇叙事式

## 段落功能速查

前奏：建立音色、氛围和第一印象。短视频 demo 里不宜太长。

主歌：交代人物、场景、关系和事件，不负责立刻爆发。

预副歌：把主歌的情绪推向副歌，通常负责升温、转折、追问。

副歌 / Hook：歌曲记忆核心，负责让听众记住主题和情绪。

桥段：提供新的视角、反转、情绪变形或最终决定。

插段：用念白、音效、新节奏或短旋律打断预期，制造新鲜感。

器乐段落：让乐器承担情绪推进，适合强调编曲和氛围。

最终副歌：通常比前面的副歌更重、更满、更像情绪总爆发。

尾奏：收束情绪，不要写成新的主歌。

## 1. 长篇叙事式

结构：前奏 → 第一主歌 → 预副歌 → 副歌 → 第二主歌 → 预副歌 → 副歌 → 桥段 → 器乐独奏 → 最终副歌 → 尾奏

核心特点：篇幅较长，适合讲完整故事。器乐独奏用于补充情绪和叙事空间。

每段功能：

- 第一主歌：交代关系起点、人物状态或故事入口。
- 预副歌：把故事推向第一个情绪疑问。
- 副歌：给出核心情绪判断。
- 第二主歌：补充新的事件或时间变化。
- 桥段：给出反转、醒悟或不能回头的决定。
- 器乐独奏：让情绪停留，不急着解释。
- 最终副歌：把核心情绪唱到最完整。

适合曲风：叙事民谣、摇滚长歌、电影主题曲、概念专辑重点曲目、华语抒情长线叙事。

适合短视频 demo 吗：不适合第一轮测歌直接上完整结构。更适合先拿副歌或桥段做短视频测试，数据好后再扩写完整版。

写作示例方向：

```text
[Verse 1]
先写第一次见面或关系已经坏掉的细节。

[Pre-Chorus]
把“我以为还能挽回”推到“其实已经来不及”。

[Chorus]
唱出主金句。

[Bridge]
承认自己终于明白，故事无法重来。
```

风险：

- 容易写太长，短视频用户等不到记忆点。
- 主歌如果只是铺陈背景，会显得慢。
- 不适合没有故事素材的用户硬写。

## 2. 洗脑病毒式

结构：副歌 → 第一主歌 → 副歌 → 第二主歌 → 副歌 → 桥段 → 副歌 → 尾奏

核心特点：副歌前置，开场就让听众记住核心句。重复密度高，传播优先。

每段功能：

- 开头副歌：直接抛出最强记忆点。
- 第一主歌：解释这句副歌发生在什么处境里。
- 第二主歌：换一个角度补充同一情绪。
- 桥段：短暂变形，避免重复疲劳。
- 最终副歌：重复并强化。

适合曲风：短视频热歌、商业流行、电音流行、广告曲、口水歌、强 Hook 情歌。

适合短视频 demo 吗：非常适合。尤其适合已经有强副歌、强口号、强重复句的作品。

写作示例方向：

```text
[Hook]
先唱最能让观众停下来的 1-2 句。

[Verse 1]
解释这句为什么成立。

[Hook]
回到同一句，但可以微调最后一个落点。
```

风险：

- 如果副歌不够强，前置会暴露短板。
- 重复太多会像广告歌。
- 情绪深度不够时，音乐平台完整版可能不耐听。

## 3. 氛围渐进式

结构：前奏 → 第一主歌 → 第二主歌 → 预副歌 → 副歌 → 桥段 → 副歌 → 尾奏

核心特点：先铺氛围和叙事，再进入副歌。情绪像慢慢升起，而不是突然爆炸。

每段功能：

- 第一主歌：建立画面和人物状态。
- 第二主歌：补充关系动作或时间变化。
- 预副歌：把低声叙事推向情绪临界点。
- 副歌：情绪第一次完整打开。
- 桥段：给出新的理解或更深的痛点。

适合曲风：独立民谣、另类 R&B、氛围流行、City Pop 慢歌、低光 emo 情歌。

适合短视频 demo 吗：适合完整歌，但不适合直接拿开头主歌做短视频。短视频应优先截副歌、预副歌末尾或最有画面的主歌句。

写作示例方向：

```text
[Verse 1]
先写一个安静但具体的画面。

[Verse 2]
加入关系里的动作。

[Pre-Chorus]
让情绪开始失控。

[Chorus]
唱出真正想说但一直没说的话。
```

风险：

- 铺垫太长会影响短视频前 5 秒。
- 适合有画面审美的用户，不适合只想快速洗脑的歌。

## 4. 能量爆发式

结构：前奏 → 第一主歌 → 副歌 → 第二主歌 → 副歌 → 桥段 → 最终副歌 → 尾奏

核心特点：主歌后直接进入副歌，转折迅速，情绪和能量输出直接。

每段功能：

- 第一主歌：用短句快速锁定处境。
- 副歌：立即爆发主情绪。
- 第二主歌：补充新的细节，不要拖慢。
- 桥段：短暂降能量或换视角。
- 最终副歌：把爆发推到最高。

适合曲风：emo 流行、旋律说唱、流行朋克、电子舞曲、热血动漫主题曲。

适合短视频 demo 吗：适合。尤其适合第一句歌词短、旋律记忆点靠前的歌。

写作示例方向：

```text
[Verse 1]
用 2-4 句把人群锁死。

[Chorus]
马上唱主金句。

[Bridge]
突然收低，说一句更私密的话。
```

风险：

- 主歌太长会拖垮爆发。
- 副歌如果旋律记忆点在句尾，观众可能等不到。
- 歌词要短句化，不然能量出不来。

## 5. 对称回旋式

结构：前奏 → A段 → B段 → 副歌 → B段 → A段 → 尾奏

核心特点：结构工整，像从一个地方出发，中间打开，再回到原点。

每段功能：

- A段：提出核心画面或命题。
- B段：换一个旋律和情绪角度展开。
- 副歌：短暂集中核心表达。
- B段回归：让听众听到情绪循环。
- A段回归：形成命运感或回环感。

适合曲风：艺术流行、另类摇滚、巴洛克流行、实验流行、概念感作品。

适合短视频 demo 吗：一般。它的完整性强，但爆点不一定早。适合做有审美感的账号，不适合急速测爆款。

写作示例方向：

```text
[A]
写一个反复出现的画面。

[B]
写这个画面背后的关系原因。

[Chorus]
用一句话点破主题。

[A Reprise]
同一画面变成新的含义。
```

风险：

- 结构美感强，但可能不够短视频。
- 对歌词能力要求高，重复必须有新含义。

## 6. 对话辩论式

结构：前奏 → 主歌A视角 → 预副歌 → 副歌 → 主歌B视角 → 预副歌 → 副歌 → 桥段 → 最终副歌 → 尾奏

核心特点：两个视角交替，形成对话、误会、争辩或自我拉扯。

每段功能：

- 主歌A：写一个人的立场。
- 预副歌：把矛盾推到副歌。
- 副歌：唱出共同问题或无法解决的痛点。
- 主歌B：写另一个人的解释、反驳或沉默。
- 桥段：让两个视角发生碰撞。
- 最终副歌：选择和解、决裂或开放结尾。

适合曲风：对唱情歌、男女双视角、内心矛盾、戏剧感流行歌、R&B 对话歌。

适合短视频 demo 吗：适合，但短视频最好只截一个视角最强的段落，或者截副歌里双方都能代入的句子。

写作示例方向：

```text
[Verse A]
我说我不是不爱。

[Verse B]
你说你只是太累。

[Bridge]
两个人终于说出真正的问题。
```

风险：

- 两个视角容易写混。
- 如果没有明确人称，听众会不知道谁在说话。
- 不适合用户只想写一个人的独白。

## 7. 悬念解决式

结构：前奏 → 第一主歌 → 预副歌 → 桥段 → 副歌 → 第二主歌 → 预副歌 → 桥段 → 副歌 → 尾奏

核心特点：先转折、先设问，再让副歌作为答案出现。

每段功能：

- 第一主歌：提出异常处境。
- 预副歌：让问题变尖锐。
- 桥段：先给出一个误导、反转或内心挣扎。
- 副歌：回答前面的悬念。
- 第二轮：重复结构，但答案更深。

适合曲风：Progressive Pop、戏剧感流行、概念情歌、带故事反转的歌曲。

适合短视频 demo 吗：中等。适合剧情型视频和反差型内容，不适合纯氛围片段。

写作示例方向：

```text
[Verse]
先写一个观众想追问的异常。

[Bridge]
给出一个“原来不是这样”的转折。

[Chorus]
唱出真正答案。
```

风险：

- 结构复杂，AI 容易写乱。
- 如果副歌答案不强，前面悬念会变成故弄玄虚。

## 8. 尾声升华式

结构：前奏 → 第一主歌 → 预副歌 → 副歌 → 第二主歌 → 预副歌 → 副歌 → 尾奏/升华段

核心特点：没有传统桥段，最后用一个全新尾声段落总结和升华。

每段功能：

- 第一主歌：建立情绪。
- 预副歌：推向副歌。
- 副歌：给出核心表达。
- 第二主歌：换细节深化。
- 尾声/升华段：不重复副歌，而是给出最终余味。

适合曲风：氛围音乐、后摇式流行、爵士流行、电影感情歌、意境型作品。

适合短视频 demo 吗：适合做完整版，不适合只测前半段。短视频可以截尾声升华段，因为它往往有最特别的句子。

写作示例方向：

```text
[Outro]
不再解释关系，只留下一个画面或一句未说出口的话。
```

风险：

- 尾声如果写成鸡汤，会毁掉前面的情绪。
- 对编曲要求高，文字不能太满。

## 9. 插段变奏式

结构：前奏 → 第一主歌 → 预副歌 → 副歌 → 插段 → 第二主歌 → 预副歌 → 副歌 → 桥段 → 最终副歌 → 尾奏

核心特点：在第一段副歌后加入短插段，制造记忆点或反差。

每段功能：

- 第一主歌：建立情绪和人群。
- 预副歌：推向副歌。
- 副歌：主记忆点。
- 插段：念白、音效、节奏变化、对话或新旋律。
- 第二主歌：带着插段后的新情绪继续。
- 桥段：最后一次变形。

适合曲风：嘻哈、旋律说唱、另类流行、实验流行、带幽默或意外转折的歌。

适合短视频 demo 吗：适合。插段如果有强记忆点，可以成为短视频传播点。

写作示例方向：

```text
[Interlude]
插入一句电话留言、语音备忘录、口头禅或反差念白。
```

风险：

- 插段不能为了新奇而新奇。
- 念白太尴尬会显得廉价。
- 插段应该服务歌曲记忆，不是打断歌曲。

## 10. 纯音乐叙事式

结构：前奏 → 第一主歌 → 器乐段落 → 副歌 → 第二主歌 → 器乐段落 → 副歌 → 尾奏

核心特点：器乐承担部分叙事功能，人声和乐器平等推进。

每段功能：

- 第一主歌：给出最少量的人声信息。
- 器乐段落：用旋律、和声或音色补充情绪。
- 副歌：让人声回到核心主题。
- 第二器乐段落：再次推进，不只是重复。
- 尾奏：让器乐收束情绪。

适合曲风：后摇、器乐摇滚、融合爵士、氛围 R&B、电影配乐感流行。

适合短视频 demo 吗：适合 BGM 和氛围号，不适合强歌词传播路线。短视频重点应放在画面和声音使用场景。

写作示例方向：

```text
[Instrumental]
用吉他、钢琴、合成器或采样继续说话，不强行塞歌词。
```

风险：

- 歌词太多会破坏器乐叙事。
- 需要更强编曲能力。
- 如果目标是求歌名或求完整版，可能不如强 Hook 结构直接。

## 给用户的选项展示格式

当用户要求扩写完整歌词时，优先用下面的简化版本展示：

```text
你现在是要把主打段扩写成完整歌词。完整歌最好先选结构，不然容易写成一堆段落硬拼。

请选择一个结构编号：

1. 长篇叙事式：适合叙事民谣、摇滚长歌、电影主题曲；短视频先测副歌更稳。
2. 洗脑病毒式：副歌前置，适合短视频热歌和商业流行；传播最快。
3. 氛围渐进式：两段主歌铺垫，适合独立民谣、另类 R&B、氛围流行；完整版更耐听。
4. 能量爆发式：主歌后直接进副歌，适合 emo 流行、旋律说唱、电子舞曲；前 5 秒更利落。
5. 对称回旋式：结构工整，适合艺术流行、另类摇滚；审美感强但不一定最短视频。
6. 对话辩论式：双视角推进，适合对唱、内心矛盾、戏剧感情歌。
7. 悬念解决式：先转折再释放，适合戏剧感强的流行歌和剧情型内容。
8. 尾声升华式：最后用新段落升华，适合氛围音乐和情绪流淌型作品。
9. 插段变奏式：中途加入念白/音效/新旋律，适合说唱、另类流行和反差记忆点。
10. 纯音乐叙事式：器乐承担情绪推进，适合后摇、器乐摇滚、融合爵士。

你直接回复数字就行。
```

## 生成完整歌词时的最低要求

用户选定结构后，输出完整歌词必须做到：

- 每个段落都写段落元标签，例如 `[Verse 1]`、`[Pre-Chorus]`、`[Hook]`、`[Bridge]`。
- 保留用户已经确认的主打段核心句，不要擅自改掉记忆点。
- 每段承担不同功能，不要把副歌意思复制到主歌。
- 主歌负责关系和故事，副歌负责记忆点，桥段负责转折。
- 如果用于短视频测试，必须额外指出最适合截取的 8-15 秒片段。
- 如果结构不适合短视频 demo，必须提醒用户先用副歌或 Hook 测试，再做完整版。
