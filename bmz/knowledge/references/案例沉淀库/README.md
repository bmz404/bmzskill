# 公开案例沉淀库

这里用于保存 bmzskill 项目公开发布时可以一起开源的匿名案例。

普通用户安装 bmzskill 后，自己通过 `/bmz-case` 沉淀的案例，默认不写进本目录，而是写入用户本机的全局私有案例库：

```text
%USERPROFILE%/.bmzskill/案例沉淀库/
```

这份私有案例库不在 bmzskill 开源仓库内，不会随项目提交，也不会被其他用户看到。不同项目、不同新对话可以读取同一台电脑上的这份私有案例库。

只有维护者明确决定匿名公开，并主动整理进本目录的案例，才会成为公开案例。

## 保存规则

- 成功模板保存到：`成功模板/成功模板-歌曲名.md`
- 待复测模板保存到：`待复测模板/待复测模板-歌曲名.md`
- 失败/避雷案例保存到：`避雷案例/避雷案例-歌曲名.md`

案例沉淀的最小单位是单条作品，不是一次团队测试。

如果一次团队测试里有多条作品，需要分别保存成多张案例卡。

## 注意

只有已经写入本目录，或写入用户本机 `%USERPROFILE%/.bmzskill/案例沉淀库/` 的案例，才能在后续对话里被称为“已沉淀案例”。

如果案例只在聊天回答里出现，但没有生成对应 `.md` 文件，就只能算“对话内临时复盘”，不能算正式沉淀。
