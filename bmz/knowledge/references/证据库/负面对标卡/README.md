# 负面对标卡

用途：沉淀公开低数据短视频样本中的失败模式，供 `/bmz-visual`、`/bmz-benchmark`、`/bmz-preflight`、`/bmz-review`、`/bmz-publish` 做避雷判断。

证据等级：A 级公开低数据样本。

来源说明：本库来自用户提供的抖音首屏截图集合。截图只能看到公开表面数据和首屏信息，不能看到后台 2s 跳出率、5s 完播率、平均播放时长、推荐流量结构等数据。

## 使用原则

负面对标卡不是为了证明某个创作者“不行”，而是帮助 bmzskill 识别常见失败模式：

- 首屏看不出这条视频和谁有关。
- 画面没有明确主体、动作或情绪。
- 歌词只在表达情绪，没有建立具体关系。
- AI 画面、封面感、播放器感过强，缺少真实使用场景。
- 数据低，但评论区没有求完整版、求歌名、求上架、跟拍等资产信号。
- 标签和标题只在描述歌曲，没有制造观众参与理由。

## 判断边界

只能判断：

- 首屏画面是否有停留理由。
- 首屏文案钩子是否能锁定人群。
- 标题、标签、字幕是否有转化方向。
- 公开互动是否出现资产信号。
- 画面形式是否落入已知低数据风险模式。

不能判断：

- 歌曲完整听感。
- 真实播放量层级。
- 后台完播数据。
- 推荐流是否异常。
- 创作者长期账号权重。

## 反馈验证

本库默认反馈验证为：负向。

如果后续某类负面样本被改造后获得明显更好数据，可以新增“修正后案例”，并与原负面卡建立对照。
