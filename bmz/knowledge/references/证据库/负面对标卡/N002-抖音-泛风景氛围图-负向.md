# N002-抖音-泛风景氛围图-负向

反馈验证：负向  
证据等级：A 级公开低数据样本  
样本来源：用户提供的抖音首屏截图集合  

## 样本特征

- 画面主要是雪景、街景、桥、月亮、烟花、树叶、城市远景等。
- 画面可能好看，但没有明确的人物、动作、冲突或关系。
- 歌词和画面之间通常只是“氛围相似”，不是“故事互相解释”。

## 公开低数据信号

- 表面互动偏低。
- 评论区很少出现观众主动认领歌曲的信号。
- 画面不脏，但缺少停留理由。

## 可能失败原因

1. 风景太泛，任何伤感歌都能套。
2. 用户不知道这条视频是在讲一个人、一个场景，还是一张普通素材。
3. 画面没有推进，前 0-2 秒无法制造“我想继续看”的理由。
4. 歌词如果也抽象，就会变成“泛风景 + 泛情绪”。

## 避雷规则

- 风景必须被赋予关系含义：这是分手后的路、等不到人的窗、没有发出去的车站，而不是普通风景。
- 首屏需要明确人物处境，哪怕画面没有人，也要让文案补足“谁在这里发生了什么”。
- 风景素材最好有动作：雨滴流动、车窗倒退、手写字、脚步、回头、门关上。
- 如果画面无法提供人物，歌词第一句必须承担锁定人群的责任。

## 影响模块

- `/bmz-visual`：原创氛围感素材、搬运氛围感素材、AI 风景都要检查是否只是“漂亮但空”。
- `/bmz-lyrics`：为风景类画面写词时，要先写具体人群和具体事件。
- `/bmz-preflight`：风景画面若无动作、无主体、无关系钩子，应提示换首屏或加关系文案。
