# A级对标卡索引

用途：让 bmzskill 在 `/bmz-visual`、`/bmz-benchmark`、`/bmz-style`、`/bmz-preflight`、`/bmz-review` 中快速检索可参考案例。  
证据等级：A 级公开对标  
覆盖范围：A001-A030  

## 使用原则

对标卡不是让用户照抄，而是帮用户快速判断：

- 当前作品更像哪类公开案例。
- 哪些画面、歌词、曲风、发布包动作已经被市场验证过。
- 哪些动作只能学习逻辑，不能照搬表面。
- 当前作品应该重点看播放、评论、收藏、转发、原声使用量还是求完整版。

检索时优先按以下顺序匹配：

1. 画面硬条件：真人 / 不露脸 / AI / 歌词流 / 乐器 / 外景 / 剪辑。
2. 性别与人物形象：男 / 女 / 中老年 / 不出现真人。
3. 曲风与音色：情绪流行、R&B、旋律说唱、DJ硬曲、国风、英文歌、翻唱等。
4. 互动目标：评论、收藏、转发、跟拍、求完整版、原声使用量。
5. 反馈验证：正向 / 有潜力 / 负向。

## bmzskill 调用方式

- `/bmz-visual`：按用户画面条件匹配“按画面类型检索”和“按资源条件检索”。
- `/bmz-benchmark`：把用户提供的截图/录屏与快速索引表比较，判断相似模板。
- `/bmz-style`：按曲风 / 内容方向检索，寻找画面和 vocal 的匹配案例。
- `/bmz-preflight`：发布前判断当前作品是否命中某类正向案例，或是否踩中该类案例的主要风险。
- `/bmz-review`：复盘数据时按互动目标检索，判断该看播放、评论、收藏、转发、关注、原声使用量还是求完整版。
- `/bmz-case`：新增案例时，先判断应归入哪个索引分类，再决定是否补充新的分类。

## 维护规则

新增 A 级对标卡时，同步更新：

1. 快速索引表。
2. 按画面类型检索。
3. 按互动目标检索。
4. 按曲风 / 内容方向检索。
5. 按资源条件检索。

如果一个案例数据后来被证明转化弱，可以保留为 A 级案例，但必须把 `反馈验证` 改成“有潜力”或“负向”，并在索引中调整适用场景。
