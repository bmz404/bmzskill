# 发布后复盘知识库

本模块负责根据发布后数据判断下一步：换画面、改歌词 Hook、改主打段进入、换发布包、继续同方向，还是换歌。复盘不是看单一指标，而是看指标之间的关系。

## 使用边界

- 开头先提示：`已进入 bmzskill 内部流程：发布后数据复盘。`
- 可以分析后台数据截图、用户手动提供的数据、评论区截图、作品表现和发布包。
- 如果只给后台数据，没有视频画面、标题、首屏、评论区或原声使用量，必须说明模板类型判断不完整。
- 不承诺从数据里反推出准确 BPM、调性、和弦或真实平台推荐机制。
- 数据不足时可以做弱判断，但必须写明样本量风险和下一步验证方式。

## 先判断数据是否可用

优先收集：

- 播放量、曝光量、点击率。
- 2 秒跳出率、5 秒留存、完播率、平均播放时长。
- 点赞、收藏、评论、转发、关注、主页点击。
- 求完整版、求歌名、求上架、歌词摘抄、原声使用量、跟拍量。
- 标题、封面、首屏文案钩子、置顶评论、标签。

播放 < 500：样本不足，不判歌，不判画面，优先检查账号权重、发布时间、标题、封面和首帧。  
播放 500-2000：只能轻判断，谨慎换歌。  
播放 > 2000 且留存/互动结构稳定：可以进入正式复盘。

## 先判断模板类型

复盘不能直接进入点赞率、评论率、收藏率。必须先判断这条视频属于哪类模板，再决定主要看什么指标。

常见模板类型：

- 情绪流行 / emo / 失恋真人近景：重点看 2 秒、5 秒、评论、收藏、转发。
- 歌词流 / 入股号 / 求歌名：重点看评论、求完整版、求歌名、收藏、音乐平台点击；警惕评论热闹但歌曲转化弱。
- DJ 硬曲 / 纯音乐 BGM / 卡点音乐：重点看原声使用量、跟拍增长、收藏、转发、跟拍画面类型；不要用评论率判断成败。
- 音乐人身份 / 制作人展示 / 编曲流程：重点看关注率、主页点击、专业认可评论、后续作品连带播放。
- 玄学许愿 / 显化歌：重点看转发、收藏、许愿评论和复用场景；警惕内容功能强但歌曲资产弱。
- AI MV / 素材氛围 / 电影感切片：重点看 2 秒停留、收藏、转发、评论是否讨论歌、音乐平台跳转。
- 下沉段子 / 反差剧情 + 演唱：重点看 2 秒停留、评论是否从段子转向歌曲、收藏、求歌名。

## 再判断曲风类型

模板类型决定“这条视频怎么呈现”，曲风类型决定“观众应该用什么方式反馈”。两者必须分开判断。

如果用户提供了参考歌曲、曲风、歌词、prompt、音色或视频内容，必须输出：

```text
曲风类型判断：
- 最接近曲风卡 / 曲风组：
- 本曲风主指标：
- 本曲风不应过度解读的指标：
```

常见曲风类型的数据重点：

- emo 流行 / 旋律说唱 / Alternative R&B：重点看 2-5 秒留存、评论、收藏、求完整版、求歌名。不要只看点赞率。
- DJ 硬曲 / 电子卡点 / Dance Pop：重点看原声使用量、跟拍量、收藏、转发。不要用评论率判断成败。
- 甜歌 / 可爱轻快流行：重点看收藏、转发、关系标签评论、是否被用作情侣/朋友/日常视频背景。
- 民谣 / 独立民谣 / 叙事 Folk Pop：重点看长评论、收藏、关注、主页点击和复听意愿。播放爆发可能慢，不要只看首轮播放。
- 抒情流行 / 抒情摇滚 / 苦情歌：重点看收藏、评论、转发、求完整版和音乐平台跳转。
- Y2K / 复古 R&B：重点看怀旧评论、求歌名、收藏、复听和“像某个年代”的评论。
- 国风 / 古风 / 中国风 R&B：重点看收藏、转发、歌词摘抄、翻唱意愿和评论里的画面联想。
- Hip-hop / Trap / Drill：重点看完播、转发、态度评论、二创使用场景和人设记忆点。

如果模板类型和曲风类型给出相反信号，不能直接下结论。例如画面数据好但曲风主指标弱，可能是画面留住了人，歌没有被记住；曲风主指标好但画面前 2 秒差，可能是歌有潜力但视频包装挡住了传播。

## 核心阈值

- 2 秒跳出率 > 50%：优先怀疑首帧、画面主体、亮度、首屏文案钩子、平台 UI 遮挡。
- 5 秒留存 < 20%：优先怀疑第一句歌词 Hook、vocal 进入、主打段进入、旋律记忆点位置。
- 播放高但点赞、收藏、评论弱：画面或标题可能能吸引人，但歌曲/歌词共鸣不足。
- 播放低但点赞率、收藏率、评论质量好：内容可能有潜力，但首帧、标题、发布时间或分发人群没打开。
- 评论区出现求完整版、求歌名、求上架：歌曲资产可能成立，优先保留歌曲方向。
- 收藏高评论低：歌曲或画面有保存价值，但评论钩子弱。
- 评论高收藏低：话题有讨论度，但主打段记忆点或收藏理由弱。
- 转发高收藏低：内容可能是情绪/关系表达工具，但歌曲本身未必成立。

## 两轮测试机制

第一轮判断包装是否挡住内容，第二轮判断内容本身是否值得继续。

- 第一轮 2 秒弱：先换画面 / 首屏文案 / 封面，不急着换歌。
- 第一轮 5 秒弱：先改第一句歌词 / vocal 进入 / 主打段进入 / 旋律记忆点位置。
- 第二轮仍 5 秒弱且评论收藏无歌曲信号：换歌或重写 Hook。
- 留存尚可但互动弱：只换标题、首屏钩子、封面或置顶评论。
- 互动信号指向歌曲但前 2 秒差：歌有潜力但视频没承接住，先换视频包装。

## 负面样本复盘

数据低且评论区没有资产信号时，必须检查是否接近负面对标：

- AI 催泪封面感：只表达伤感，没有具体关系。
- 泛风景氛围图：画面好看但没有人物动作和故事。
- 静态封面播放器式：像歌曲发布通知，不像短视频内容。
- 暗场小字：首屏看不清，字幕/说明/平台 UI 抢焦点。
- AI 海报感：精致但没有活人感。
- 伪音乐人摆拍：有乐器设备但没有真实动作。

如果命中，必须说明命中点和下一轮避开方式。

## 判断动作

- 换画面：2 秒跳出高、首帧主体弱、标题/画面不一致、评论区只夸画面不问歌。
- 改歌词 Hook：5 秒留存低、第一句太长、记忆点靠后、目标听众不清。
- 改发布包：留存尚可但互动弱；标题、首屏文案、置顶评论没有给用户评论理由；标签过散或打错人群。
- 继续放大：2 秒跳出可控，5 秒承接不错，评论区有求完整版/求歌名/求上架/共鸣故事，收藏、转发或原声使用出现明显信号。
- 先不投流：没有自然数据证明歌曲资产成立前，不建议用投流掩盖内容问题。
