# 新手引导知识库

本模块负责把新用户带进 bmzskill 的最小闭环。新手引导不能只介绍功能，要让用户知道：bmzskill 不是爆款预测器，而是“假设生成 + 发布前判断 + 数据复盘 + 迭代决策”的音乐短视频测试系统。

## 使用边界

- 开头先提示：`已进入 bmzskill 内部流程：新手引导。`
- 不承诺爆款、收益、涨粉、上架或版权变现成功。
- 不要求用户先理解所有模块；用问题把用户带到下一步。
- 不让用户自己选择复杂模块名，最终都用 `/bmz` 统一入口。

## 必须讲清楚的一句话

如果你是新手，先不要做完整专辑、完整 MV 或复杂账号规划。先跑一个最小闭环：用最低成本做出一个 20-30 秒主打段 demo，发布到短视频平台，看真实数据，再决定下一步。

## 完整路径

必须先展示这条路径：

```text
定位 / 受众
→ 主题 / 情绪
→ 8 句主打段歌词
→ 歌词质检
→ 曲风 / vocal / prompt
→ 视频画面
→ 发布前体检
→ 发布包
→ 发布后数据复盘
→ 下一轮迭代
```

解释时要强调：短视频里先测的不是完整歌，而是最能打的 8 句左右主打段。它负责让用户在 2-5 秒内产生关联。

## 新手必须理解的底层逻辑

1. 不是先问“这首歌好不好听”，而是先问“它写给谁”。
2. 不是先做完整作品，而是先做可测试片段。
3. 不是用感觉判断，而是用发布数据判断下一步。
4. 不是一次性改所有东西，而是每轮只改一个核心变量。
5. 不是追求所有人喜欢，而是先锁定一类人。

## 最小闭环

推荐新手从以下材料开始：

- 一个想写的主题。
- 一句最想表达的话。
- 一个目标听众。
- 一个短视频画面想法。
- 一段现有歌词或 demo。

如果用户什么都没有，给 3 个入口选项：

1. 从主题开始：`/bmz 我想写一首关于......的歌`
2. 从歌词开始：`/bmz 检查这段歌词`
3. 从视频开始：`/bmz 检查这条待发布视频`

## 新手常见误区

- 一开始就写完整歌，结果不知道哪 8 句能打。
- 用漂亮词堆情绪，但没有具体关系和动作。
- 画面好看但没有人物、故事或停留理由。
- 标题、首屏文案、歌词和标签互相抢焦点。
- 数据一差就换歌，没有先判断是画面、Hook、发布包还是分发问题。
- 只看点赞，不看收藏、评论、转发、求歌名、求完整版和原声使用。

## 引导问题

新手引导结尾必须主动问用户：

```text
你现在更接近哪一种？
1. 我只有一个主题，还没写歌词。
2. 我已经有歌词或主打段。
3. 我已经有视频，准备发布。
4. 我已经发布了，有后台数据。
```

然后给对应下一步：

- 选 1：`/bmz 开始写歌，主题为......`
- 选 2：`/bmz 检查这段歌词`
- 选 3：`/bmz 检查这条待发布视频`
- 选 4：`/bmz 复盘这条作品数据`
