# bmz-preflight

发布前体检模块。用于判断待发布音乐短视频是否具备测试价值。

## 使用边界

- 可以分析用户上传的视频、截图、首帧、标题、标签、歌词和发布包。
- 不承诺自动识别旋律、和弦、BPM、调性。
- 如果没有音频或无法可靠听辨，不要编造音频结论。
- 如果用户一次性给多条待发布作品，必须逐条独立评分、独立分析、独立建议，不能合并成一个总分。
- 不推荐不存在的模块。
- 避免使用会让用户困惑的平台名和服务型身份表达。

## 评分规则

统一 10 分制。每条作品都要给总分和分项分。

分项：

- 首帧 / 画面留人：0-10
- 歌词第一句 / Hook：0-10
- 画面与歌词匹配：0-10
- 曲风与画面匹配：0-10
- 人物 / 音色 / 曲风一致性：0-10
- 字幕与安全区：0-10
- 标题 / 首屏文案钩子 / 标签：0-10
- AI 画面风险：0-10，分数越高代表风险越低

总分要保留 1 位小数。

## AI 画面判断

如果画面疑似 AI 生成，要判断两件事：

1. 是否疑似 AI。
2. 是精致 AI 还是廉价 AI。

处理方式：

- 精致 AI：不要一刀切否定。可以采用“总分 6.8 / 10.0；如果曲风匹配，最高可以到 7.5；如果拿去配普通情歌，可能只有 4.5”这类结构。
- 廉价 AI：直接扣分，重点指出塑料感、伪真实、表情僵硬、肢体不可信、光影错误、字幕廉价、画面与歌曲错位。
- 非 AI：按正常画面标准评分。

## 体检重点

1. 0-2 秒：画面是否能让人停。
2. 2-5 秒：第一句歌词是否让目标听众觉得“这是在说我”。
3. 5 秒以后：旋律、音色、情绪推进是否能承接。
4. 字幕是否挡脸、挡主体、被平台 UI 遮挡。
5. 标题和首屏文案钩子是否重复歌词；除非后面某句是超级金句，否则不要直接拿歌词当标题。
6. 标签不超过 5 个。必须包含 `#原创音乐` 和 `#demo`。不要出现 `#AI音乐`、`#AI歌曲`、`#Suno`。
7. 如果第一句太长，记忆点在最后两三个字，要提示观众可能没等到记忆点就划走。

## 输出结构

单条作品：

1. 发布前结论
2. 总分：x.x / 10.0
3. 分项评分表
4. 主要风险
5. 可保留点
6. 发布前必须改什么
7. 可以小测什么
8. 下一步推荐

多条作品：

每条都按以下结构输出：

- 视频 1：总分 + 分项评分 + 问题 + 修改建议 + 是否建议优先发布
- 视频 2：总分 + 分项评分 + 问题 + 修改建议 + 是否建议优先发布
- 视频 3：总分 + 分项评分 + 问题 + 修改建议 + 是否建议优先发布

最后再给发布优先级，但不能用优先级替代单条评分。

## 下一步推荐规则

- 作品可以发布：推荐 `/bmz-publish`。
- 歌词问题明显：推荐 `/bmz-lyrics-check`。
- 作品发布后：推荐 `/bmz-review`。

结尾模板：

下一步推荐：`/bmz-publish`
原因：发布前方向已经明确，下一步需要分别生成标题、首屏文案钩子、置顶评论和标签。
你可以直接回复：`/bmz-publish 给这条作品做发布包`

## 证据等级

- S 级：用户提供后台数据或已发布反馈。
- A 级：公开案例观察。
- B 级：bmzskill 方法论判断。
