# Changelog

## 0.2.2

- 保持 `/bmz` 为唯一公开入口。
- 新增 `skills/bmz/workflows/song-release-loop.md`，用于描述从主题、歌词、发布前检查、发布包到数据复盘的完整闭环。
- `/bmz/SKILL.md` 新增内部工作流说明：连续项目推进时先参考 workflow，再调用对应模板。
- 安装包仍只包含 `bmz/` 一个 skill 文件夹。

## 0.2.1

- 保持 `/bmz` 为唯一公开入口。
- 将 `/bmz` 主入口精简为分诊与统一回复层。
- 新增 `skills/bmz/templates/`，把创作启发、歌词质检、发布前体检、发布包生成、发布后复盘拆成内部模板文件。
- 增加旧入口兼容提示：用户输入旧入口时只提示“已进入 bmzskill 内部流程：xxx”，不解释版本变化。
- 安装包仍只包含 `bmz/` 一个 skill 文件夹。

## 0.2.0

- 将 bmzskill 从 5 个公开入口收拢为 1 个总控 skill：`/bmz`。
- 把歌词质检、发布前体检、发布包生成和发布后复盘规则合并进 `/bmz/SKILL.md`，由 `/bmz` 统一识别需求并回复。
- 降低跨平台使用门槛：其他 Agent 不需要理解多个 slash 入口或子 skill 调度。
- 安装包改为只包含 `bmz/SKILL.md`，用户安装后只需要记住 `/bmz`。

## 0.1.3

- 新增面向普通用户的安装包说明，区分源码包和可安装 skill 包。
- 明确 GitHub Code ZIP 会多一层仓库目录，不能直接作为 Codex skills 目录使用。
- 推荐使用 `bmzskill-codex-skills-v0.1.3.zip`，解压后顶层直接是 5 个 skill 文件夹。
- 保持 5 个 skill 的功能规则不变，本版本只修复安装与打包体验。

## 0.1.2

- 将 `/bmz 开始写歌` 调整为创作启发入口，默认给方向、听众分析和结构骨架，不直接代写完整 8 句。
- 增加目标听众分析要求，先判断主题写给谁、对方处在什么关系状态，以及为什么会在短视频里停留。
- 重写 8 句主打段结构说明方式，按 1-4 上半段副歌、5-8 下半段副歌讲解每句功能和情绪递进。
- 增加高危 AI 模板意象黑名单，禁止主动推荐常见模板场景和动作。
- 保留用户明确要求时可输出歌词草稿的边界，并要求写完后回到歌词质检。

## 0.1.1

- 移除发布版中与未发布能力相关的入口表述。
- 收紧发布前体检边界，明确上传视频后的判断需以用户提供信息为准。
- 调整歌词质检措辞，弱化“AI 味”表达，改为关注缺乏活人感等具体问题。
- 明确歌词质检默认不主动改写；仅在用户明确要求时提供修改版。
- 加强逐句审查要求，让每组歌词给出更具体的风险解释和手动修改方向。
- 同步 README、许可证和 docs 目录结构。

## 0.1.0

- 建立 bmzskill 发布包。
- 提供主入口、歌词质检、发布前体检、发布包生成和数据复盘五个核心能力。
- 建立公开说明文档、使用路径和基础判断规则。
