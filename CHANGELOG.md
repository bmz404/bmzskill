# Changelog

## 0.3.5

- 修复歌词质检中“已被市场验证的现有作品”被按素人原创草稿低分否定的问题：确认现有大流量作品时，先承认公开市场信号，再分析可学习机制、不可复刻边界和版权风险。
- 为现有大流量作品增加评分口径：传播评分原则上不低于 7.0，歌词评分原则上不低于 6.5，除非能明确说明其流量主要来自歌手体量、平台包装、热点事件或特定历史语境。
- 强化创作启发输出：默认给出 3 种有标题区分的创作方向，每个方向显式包含【听众与具体处境】和【情绪张力与未完成瞬间】。
- 强化 8 句主打段规则：第 1-2 句必须是主金句入口，短、直、扎心、低理解成本，并至少出现“你 / 我 / 我们 / 他 / 她 / 爱”中的任意一个代入词；第 7-8 句只承担未完成感、评论口或回味点。
- 在创作启发的推荐方向之后显式输出【主打段测试假设】，让用户知道本轮主打段具体测试什么。
- 恢复并保留 `knowledge/case-loading.md`，确保所有专家子 Agent 按案例库加载协议读取完整方法论、方法卡、A 级对标卡和歌词样本。

## 0.3.4

- 统一歌词相关流程的结尾推荐：涉及歌词创作、歌词质检和歌词迭代时，下一步优先推荐 `/bmz 为我提供下一版歌词的改进方向`。
- 移除旧的“复查我修改后的歌词 / 检查我的 8 句主打段”等容易把流程带回代写或成品歌词生成的推荐入口。
- 更新主入口规则，明确歌词相关下一步不得推荐“帮我改写歌词”“帮我写 8 句主打段”等诱导代写的入口。

## 0.3.3

- 收紧 `songwriting` 创作启发边界：禁止直接替用户写完整歌词、完整 8 句主打段或可直接发布的成品句子。
- 即使用户要求“直接帮我写一版 / 给我 8 句 / 改成 8 句主打段”，也只能输出创作方向、8 句功能骨架、每句任务、可选表达角度、关键词池、避雷清单和自检标准。
- 更新 `songwriting` 输出格式，将“8 句主打段草稿”改为“8 句主打段功能骨架”，避免 Agent 越界代写歌词。

## 0.3.2

- 新增全局硬约束：所有功能模块的判断标准必须基于 `knowledge/references/` 中的真实案例库、负面对标库、曲风库、歌词库、完整方法论和 B 级方法卡，不得只凭模型常识或摘要规则输出。
- 新增全局输出深度下限：除非用户明确要求“简版 / 摘要 / 只给结论”，任何功能模块都禁止输出摘要，必须保留案例依据、方法论依据、关键拆解、风险、评分或判断过程、最小修改动作和下一步建议。
- 强化 `/bmz` 主入口转述规则：主 Agent 统一回复时不得压缩专家子 Agent 的完整深度答案。
- 强化视频画面判断优先级：视频审查必须首先检查是否为廉价 AI 生成素材，再判断画面承载、曲风匹配和审美。
- 新增廉价 AI 总分封顶规则：若判定为“廉价 AI 伪真实素材”，无论画面与曲风是否匹配、画面是否漂亮，整体视频作品总分不得超过 4.0 / 10.0；精致 AI / 高级风格化 AI 继续按正常评分标准判断。

- `knowledge/publish.md` 恢复发布包深度：发布目标判断、A/B/C 三套测试方案、分目标写法、标题/首屏/置顶评论分工、标签规则、测试假设和发布后观察指标。
- `knowledge/songwriting.md` 恢复创作启发深度：目标听众、具体处境、欲望/阻碍、伪装/禁忌感、未完成瞬间、8 句主打段结构、完整歌词扩写和元标签规则。
- `knowledge/onboarding.md` 恢复新手引导深度：最小闭环、完整路径、新手误区、材料补充项和四选一分流。
- `templates/*.prompt.md` 增加深度保护规则：除非用户明确要求简版，否则子 Agent 必须完整执行对应知识库输出格式，不得压缩成摘要。
- 将根目录完整 `知识库/` 迁移到 `skills/bmz/knowledge/references/`，让公开单 skill 自包含完整方法论、FAQ、A级对标卡、B级方法卡、负面对标卡、曲风库、好歌词案例库、坏歌词样本库和高频风险词表。
- 新增 `knowledge/case-loading.md` 作为统一案例库加载协议，要求所有专家子 Agent 先读索引、再读具体卡片；深度审查必须至少读取 1-3 张最相关案例卡/方法卡/曲风卡/歌词样本。
- 6 个 `templates/*.prompt.md` 全部接入 `knowledge/case-loading.md`，确保判断基于完整案例库和完整方法论，而不是只基于专家摘要文件。

## 0.3.0

- 明确采用「主 Agent + 子 Agent」架构，修复长会话记忆压缩导致的专家判断质量下降。
- 主 Agent（`/bmz/SKILL.md`）改为纯客户经理：只做意图识别、填模板字段、spawn 全新子 Agent、统一转述；自身不再执行专家判断，也不把专家规则堆在自己上下文里。
- 将专家流程拆为两部分：`templates/*.prompt.md` 是带 `{{USER_INPUT}}` / `{{CONTEXT}}` 占位符的参数化提示词模板；`knowledge/*.md` 是独立的专家知识库文件，由子 Agent 自行加载。
- 每次派发的子 Agent 都是独立、干净的新实例，知识库从头满血加载，长会话不会压缩任何一次判断。
- `workflows/song-release-loop.md` 同步改为「每个阶段 spawn 对应子 Agent」的编排说明。
- 安装包仍为单 skill `bmz/`，用户只需记住 `/bmz` 一个入口。
